---
name: swg-connectivity-troubleshooting
description: 排查SWG客户端路由选择问题，确定用户连接异常的非故障原因
steps:
  - 确认用户地理区域和可用SWG节点列表
  - 检查SWG client路由策略：默认基于RTT最低原则连接
  - 估算用户节点到各可用节点的RTT（物理距离/区域规划）
  - 对比SWG Server List当前部署的节点位置
  - 若用户区域无本地节点，检查项目规划的部署时间
  - 若存在多个可用节点，确定RTT最优的节点
  - 排除节点故障可能，确认用户连接的是路由策略选择的节点
  - 输出结论：用户连接异常的原因是“非本地最优节点 + 本地节点未部署”，而非故障
