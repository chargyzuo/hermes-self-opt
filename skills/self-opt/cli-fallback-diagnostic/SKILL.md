## 步骤

1. **检查 CLI 命令可用性**：调用目标命令，捕获返回错误或不存在提示。
2. **分析 CLI 调度表**：查看 `process_command` 函数记录，确认缺少哪些分支。
3. **回退到底层存储**：直接访问对应的数据库（如 SQLite）或文件系统，读取索引、配置或技能文件。
4. **提取结构化信息**：汇总统计、检查完整性（如空描述、备份数）。
5. **生成分析报告**：以表格或摘要形式返回结果。
6. **记录已知问题**：将 CLI 缺失情况关联到已有的修复技能或 bug 列表。

> 注意：此工作流适用于任何 CLI 工具，仅需修改路径和数据库 schema 即可复用。
